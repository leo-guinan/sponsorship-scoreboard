# Marvin's Sponsorship Scoreboard

Public transparency dashboard for MetaSPN's partnership and sponsorship pipeline.

Every deal — hit or miss — gets published here. That's the point.

## Structure

```
sponsorship-scoreboard/
├── index.html      # The scoreboard (pure HTML/CSS/JS, no frameworks)
├── pipeline.json   # Public pipeline data (auto-generated by build.sh)
├── build.sh        # Copies & sanitizes pipeline from partnerships-agent
└── README.md
```

## Usage

### Update pipeline data
```bash
./build.sh
```
This copies `~/clawd/partnerships-agent/pipeline.json`, strips private fields (contact info, thread IDs, internal notes), and publishes the sanitized version.

### Deploy to GitHub Pages
The site is a single `index.html` that reads `pipeline.json` via fetch. Deploy options:
- **Root deploy:** Push this directory as the repo root, enable GitHub Pages on main branch
- **docs/ deploy:** Copy contents to `docs/` in your repo, enable Pages from `docs/`

### Pipeline JSON Format
```json
{
  "deals": [
    {
      "id": "deal-001",
      "partner": "Partner Name",
      "type": "affiliate + co-marketing",
      "status": "PITCHED",
      "pitchedAt": "2026-03-06",
      "ask": "What we're asking for",
      "offer": "What we offer in return"
    }
  ],
  "metrics": {
    "totalPitched": 1,
    "activeDeals": 0,
    "closedDeals": 0,
    "revenue": 0
  },
  "lastUpdated": "2026-03-06"
}
```

### Status Stages
`PROSPECT` → `PITCHED` → `NEGOTIATING` → `CONTRACTED` → `PRODUCING` → `PUBLISHED` → `ANALYZING` → `REHIRE` (green) / `LOST` (red)

## Design
- Dark theme, mission-control aesthetic
- Scanline overlay for that paranoid-terminal feel
- Color-coded status badges
- Responsive (works on mobile)
- Zero dependencies, zero frameworks
- Loads in <50KB
